# Floaty 项目 Code Review

请对当前 git 变更进行全面的代码审查。按照以下清单逐项检查，给出具体的问题和改进建议。

## 审查步骤

1. 先运行 `git diff --staged` 和 `git diff` 查看所有变更
2. 按下面的清单逐项审查
3. 输出审查报告，包含问题严重级别（🔴 严重 / 🟡 警告 / 🔵 建议）

## Rust 后端审查清单

- [ ] **Tauri Command 规范**: 新 command 是否在 `lib.rs` 的 `invoke_handler` 中注册？
- [ ] **update_note 返回值**: 是否保持返回 `()`？绝不返回 Note 对象
- [ ] **数据库操作**: 是否通过 `AppState` 获取连接？是否处理了 SQL 错误？
- [ ] **错误处理**: 是否使用了合适的 Result 类型？错误信息是否有意义？
- [ ] **窗口管理**: 新窗口是否初始隐藏？是否处理了 DPI 缩放？
- [ ] **Clippy**: 运行 `cargo clippy` 是否有警告？
- [ ] **格式化**: 运行 `cargo fmt --check` 是否通过？

## React 前端审查清单

- [ ] **状态管理**: 是否遵循了 `useState` + 直接 `invoke()` 的模式？（无全局状态库）
- [ ] **TipTap 编辑器**: 是否保持了编辑器作为内容唯一源的原则？
- [ ] **Tailwind 样式**: 是否使用了 Tailwind CSS 4 语法？编辑器内列表样式是否正确？
- [ ] **shadcn/ui**: 是否使用了 `@/components/ui/` 路径？是否符合 New York 风格？
- [ ] **TypeScript**: 是否有 `any` 类型？类型定义是否完整？
- [ ] **窗口 API**: 拖拽是否使用 `startDragging()`？物理像素是否转换为逻辑像素？
- [ ] **路径别名**: import 是否使用 `@/` 别名？

## 通用审查清单

- [ ] **安全性**: 是否存在 XSS、SQL 注入等安全风险？
- [ ] **性能**: 是否有不必要的重渲染？是否有 N+1 查询？
- [ ] **错误处理**: 是否有未捕获的异常？用户是否能看到有意义的错误信息？
- [ ] **代码风格**: 命名是否一致？是否有死代码？

## 输出格式

```
## Code Review 报告

### 概览
- 变更文件数: X
- 变更类型: [新功能/Bug修复/重构/...]

### 问题列表
🔴 [严重] 文件:行号 - 问题描述
🟡 [警告] 文件:行号 - 问题描述
🔵 [建议] 文件:行号 - 问题描述

### 总结
[一句话总结审查结果和主要建议]
```
